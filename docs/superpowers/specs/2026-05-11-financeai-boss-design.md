# 中小企业老板财务AI助手 - 技术设计文档

## 1. 项目概述

### 1.1 项目名称
**FinanceAI Boss** - 中小企业老板财务AI助手

### 1.2 核心价值
让不懂财务的中小企业老板，通过自然语言对话，一句话了解公司经营状况，获取AI驱动的决策建议。

### 1.3 目标用户画像
- 中小企业老板/管理层
- 不具备专业财务知识
- 时间宝贵，需要快速获取关键信息
- 主要在PC端使用，偶尔移动端查看

### 1.4 数据安全承诺
- Excel/CSV文件在浏览器本地解析
- 数据仅本地处理，不上传第三方
- 所有AI分析在用户本地或可信环境执行

---

## 2. 功能范围

### 2.1 核心功能（MVP）

| 功能模块 | 具体功能 |
|----------|----------|
| 数据上传 | 支持Excel(.xlsx/.xls)和CSV文件上传，自动识别财务报表结构 |
| 经营全景 | 展示收入、成本、利润核心指标，支持月度/季度/年度切换 |
| 现金流健康度 | 收支趋势图、账期预警、现金流预测 |
| 成本异常预警 | 自动识别异常波动，定位问题科目 |
| AI对话助手 | 自然语言查询财务数据，深入追问分析 |
| 决策建议 | 基于数据洞察，给出可执行的建议 |

### 2.2 交互方式
- **仪表盘**：可视化图表展示核心指标
- **对话窗口**：自然语言追问，深入分析
- **预警卡片**：成本异常、账期到期等主动提醒

## 3. 技术架构

### 3.1 技术栈选择

| 前端框架 | React 18 + TypeScript | 生态成熟，组件丰富 |

## 5. 后端设计

### 5.1 API接口设计

#### 5.1.1 文件上传与解析
```
POST /api/v1/upload
Request: multipart/form-data { file: File }
Response: {
  "success": true,
  "data": {
    "filename": "财务报表.xlsx",
    "sheets": ["利润表", "资产负债表", "现金流量表"],
    "record_count": 156,
    "date_range": { "start": "2024-01", "end": "2024-03" }
  }
}
```

#### 5.1.2 获取分析数据
```
POST /api/v1/analyze/overview
Request: {
  "date_range": { "start": "2024-01", "end": "2024-03" },
  "granularity": "monthly"  // monthly / quarterly
}
Response: {
  "revenue": 125000,
  "cost": 89000,
  "profit": 36000,
  "profit_margin": 0.288,
  "trend": [...]
}
```

```
POST /api/v1/chat
Request: {

  "reply": "经过分析，这个月成本上涨的主要原因...",
```

### 5.2 数据模型

#### 5.2.1 财务记录
```python
class FinancialRecord(BaseModel):
    id: str
    date: date
    type: Literal["income", "expense"]
    category: str  # 收入/成本类别
    amount: Decimal
    description: str
    counterparty: str | None  # 对方单位
    payment_status: Literal["paid", "pending", "overdue"]
```

#### 5.2.2 分析结果
```python
class AnalysisResult(BaseModel):
    revenue: Decimal
    cost: Decimal
    profit: Decimal
    profit_margin: float
    cost_breakdown: dict[str, Decimal]
    trends: list[TrendPoint]
    alerts: list[Alert]
```

### 5.3 核心算法

#### 5.3.1 成本异常检测
- 使用Z-Score算法检测异常值
- 设定阈值为2.5（超过2.5个标准差标记为异常）
- 支持同比、环比两种比较方式

#### 5.3.2 现金流预测
- 基于历史数据的移动平均法
- 考虑季节性因素的指数平滑法
- 输出未来3个月的预测区间

#### 5.3.3 AI提示词设计
```
系统提示词：
你是一位专业的中小企业财务顾问。用户是中小企业老板，
不具备专业财务知识。请用通俗易懂的语言解释财务问题，
给出具体、可执行的建议，避免使用专业术语。

分析时需要：
1. 结合具体数字说明
2. 给出明确结论
3. 提供可操作的建议
4. 如有风险，明确指出
```

## 6. AI能力设计

### 6.1 核心对话场景

| 场景 | 用户问题示例 | AI回复要点 |
|------|-------------|-------------|
| 经营概览 | "我这个月赚了多少？" | 收入、成本、利润同比环比 |
| 成本分析 | "成本为什么上涨了？" | 定位具体科目，给出原因 |
| 现金流 | "账上还有多少钱？" | 当前余额+未来预测 |
| 风险预警 | "有什么需要注意的？" | 异常项+建议措施 |
| 决策支持 | "要不要接这个订单？" | 成本核算+利润分析+风险提示 |
| 对比分析 | "和上个月比怎么样？" | 关键指标对比+变化原因 |

### 6.2 数据洞察生成
- 自动识别数据中的规律和异常
- 生成结构化的分析报告
- 提供置信度和数据来源

## 7. 安全设计

### 7.1 数据安全
- Excel文件在浏览器端使用JavaScript解析（SheetJS库）
- 数据不经过后端服务器
- 使用IndexedDB存储解析后的数据
- 支持数据一键清除

### 7.2 API安全
- 后端API添加请求频率限制
- AI模型调用添加内容过滤
- 敏感操作添加日志记录

## 8. 部署方案

### 8.1 开发环境
- Node.js 18+
- Python 3.11+
- pnpm / npm

### 8.2 生产环境
- 前端：静态托管（Vercel/Netlify）或 nginx
- 后端：Docker容器化部署
- AI服务：OpenAI API 或 自托管模型

### 8.3 快速启动
```bash
# 前端
cd frontend && pnpm install && pnpm dev

# 后端
cd backend && pip install -r requirements.txt && uvicorn main:app --reload
```
